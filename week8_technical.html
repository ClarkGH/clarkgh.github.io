Regular Expressions

5/1/2014

     Before I get into my explanation of Regular Expressions, I would like to refer to this fantastic resource that I used to work with regular expressions. If there is still some confusion with regular expressions after reading this entry, I recommend reviewing the wikipedia article as well as this link explaining regular expressions. I recommend utilizing the the RegEx quickstart and tutorial on the last link provided if you have no understanding of them, because RegExp is most definitely a very deep subject and I will only cover the basics.

     Regular expressions or RegEx are sequences of characters used to search via pattern comparisons with strings. In programming, they are used in "find and/or replace" certain patterns found. The great thing about regular expressions is that the code is univeral, and once you understand them, you can utilize them with a large amount of programming languages. What could take up lines of code may end up only taking one line with RegEx.

For the purposes of this entry, I recommend clicking here and playing with RegExp. I'll list off quite a few of the many different tokens used in searching through RegEx. Know that each token helps you search through different things with variations depending on combinations you make.

  1. The Dot "."

    2. The dot matches any character that isn't a line break.

  2. [\s\S] or [^]

    3. [^] isn't supported in all browsers. This can be used to match any character, including line breaks.

  3. [\w]

    4. This will match any low-ascii characters (anything non-accented).

  4. [\W]

    5. This will match any character that is not a word character.

  5. [\d]

    6. This will match any numeric character (0-9).

  6. ^

    7. This matches the beginning of the position of the string or line (if multiline is enabled).

  7. $

    8. This matches the end position of the string or long (again with multiline being enabled)

  8. \b

    9. This will match the position of a word boundary (whitespace, punctuation, or the start or end of the string)

  9. \B

    10. This will match the position that is not a word boundary.

  10. \000 - \377

    11. This will search through any octal escaped character (a special character) that has a value less than (\377).

  11. \x00 - \xFF

    12. This will search for any Hex escaped character (similar to octal).

  12. \u0000 - \uFFFF

    13. This will match any unicode escaped character.

  13. \cA - \cZ

    14. This will match any ctrl character.

  14. \t

    15. This will match any tab character

  15. ()

    16. This groups multiple tokens together and makes a capture group for extracting a substring or using a back reference.

  16. /1 - /anynumber

    17. This will show the results of a previous capture group. Review the reference for further understanding.

  17. (?:)

    18. This groups multiple tokens together while not creating a capture group

  18. (?=)

    19. This will match a group after the main expression while not including the result.

  19. (?!)

    20. This will search through every group that does not match.

  20. +

    21. This will match through one or more of the preceding token.

  21. *

    22. This will match through 0 or more of the preceding token.

  22. {#,#}}

    23. This matches through the specific quantity given in the curly braces. {1,2} matches 1 to 2, {1} will match just 1, and {2,} will match two or more.

  23. ?

    24. If after a token this will match 0 or one of the preceding token, making it optional. If after a quantifier, this will match as few characters as possible as opposed to how quantifiers generally match as many as possible.

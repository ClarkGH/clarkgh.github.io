Git Cheatsheet

5/2/2014

     For my own benefit as well as the benefit of those who are learning how to work with git and Github through the terminal, I'm going to post a few shortcuts with explanations and utilize this page as a cheat sheet for more than just Git. Make sure you have git installed first, if git is not installed click here and then follow the steps here. For all of the below commands, make sure you're in the correct repository's directory.

    

  1. Check if Git is Installed and/or Git Version

    2. git --version


  2. Push to Github:

    3. Create a new repository on GitHub and change to the correct directory in the terminal.

    git init (which will initialize a repository in the directory)


  3. Create a New File:

    4. Readme.txt

    touch index.html style.css Readme.txt


  4. View the Repository's Status:

    5. git status


  5. Add Files to Repository:

    6. git add [filename]

    git add . (this will add all files modified in the repository)


  6. Commit repository

    7. git commit -m "message"/i>


  7. Change the Repository URL:

    8. git remote add origin https://github.com/username/repo.git


  8. Send Commits to Master Branch:

    9. git push -u origin master

    git push origin master


  9. Checking Current Remote:

    10. git remote -v


  10. Set a New Upstream Remote:

    11. git remote add upstream https://repositoryurl.git


  11. Checking Current Remote:

    12. git remote -v


  12. Fetch Changes from Upstream Remote:

    13. git fetch upstream


  13. Check Upstream Master Branch:

    14. git branch -va


  14. Check the Branch You're On:

    15. git branch


  15. Merge the Upstream Master Into Local Repo:

    16. git merge upstream/master

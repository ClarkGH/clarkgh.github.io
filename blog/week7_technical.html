SQL Injection Hacks and Prevention

     SQL injections are a dangerous possibility with databases where data can both be stolen or be destroyed. Malicious users generally 'inject' SQL commands into SQL statements through webpage input that can alter SQL statements. For the purposes of this, let's say we have a database with protected user information. In the case of SELECT * FROM users WHERE userid = 242 OR 1=1, someone could bypass any password information with an SQL injection and access all of their information. Since the code is valid with unprotected SQL sites, the hacker would receive all of the information since WHERE 1=1 is always returned as true. To summarize, a hacker could get access to database information by inputting OR =.

     There is more that hackers can do with SQL injections, like deleting all of the table's data, but beyond that there are some ways to protect a site from SQL injections. The best way to protect yourself from SQL injections is by using parameters. Parameters are all checked to ensure that they are correct and are not treated as part of the SQL to be executed. This prevents hackers from being able to access your website data. Good examples for preventing SQL Database injection hacks can be found here.

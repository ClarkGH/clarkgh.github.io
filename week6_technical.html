Functional Programming

     Functional programming languages allow you to write functions that take n arguments and return a value (mathematical functions). A completely functional program will always yield the same value for its input. When the program is executed, the expression is evaluated. Functional programs require the functions to be first-class, which means they will be treated like any value as well that the possibility of defining and manipulating of other functions remains.


Ex. function(x,y) {return x+y;}


     Object-Oriented languages focus on control flow and use both statements as well as expressions. Almost everything is an object in OOP. Data tends to carry a collection of functions as opposed to functional programs where data and functions have more in common with eachother. OOP makes it easier to plan and create complex systems since it gives devs the ability to break the system down into objects.


Ex. var Dance = function(type)

{ this.Type = type}


Functional programs are core based on mathematical function composition. Functional programs are generally shorter and easier to understand than Object-Oriented Programs. Functional programs do not use statements, focus on Data Flow and fans of Functional programming believe that it is faster and more productive than OOP. Regardless of what people say, functional programming is all interwoven. If you change one program it will break many that are interwoven with it, making it harder to manage than OOP.
